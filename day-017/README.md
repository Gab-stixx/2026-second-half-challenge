## Training: Introduction to Trees

### Learned Today

#### Tree Basics
- **Node:** Contains data and pointers to children
- **Root:** Top node with no parent
- **Leaf:** Node with no children
- **Edge:** Connection between nodes
- **Height:** Distance from node to deepest leaf
- **Depth:** Distance from root to node

#### Binary Trees
- Each node has maximum 2 children (left, right)
- Not necessarily sorted or balanced
- Foundation for BST, AVL, Red-Black trees

#### Key Concepts
- Parent-child relationships
- Subtrees (left subtree, right subtree)
- Tree traversal (DFS, BFS - coming next)
- Level order (depth-based layers)

### Why Trees Matter
- **Hierarchical data:** File systems, DOM, organizations
- **Search:** Binary Search Trees for O(log n) operations
- **Foundation:** Before learning graphs, heaps, tries

### Next Steps
- Tree traversal methods (inorder, preorder, postorder)
- Implement traversals recursively
- LeetCode tree problems

### Reflection
Data structures progression is clear:
1. Arrays - sequential, O(n) search
2. Linked Lists - dynamic, still O(n) search
3. Trees - hierarchical, O(log n) possible with BST
4. Graphs - networks, complex relationships

Each layer adds complexity and capability.
